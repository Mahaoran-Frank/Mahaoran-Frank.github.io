# Coding Agent：谁掌握它，谁掌握未来的生产侧

> 写于 2026-07-10 | 

---

## 一件看起来很小的事

2026年7月，OpenAI 宣布将独立的 Codex 应用并入 ChatGPT 桌面版。新版 ChatGPT 分成了三个模块：Chat（日常对话）、Work（办公与研究）、Codex（软件开发）。

很多用户的第一反应是：怎么又改版了？额度还公用？

但如果只把这件事当成一次产品迭代，就错过了它背后更值得看清楚的东西。

---

## 2026年初：AI 公司面临的一个根本选择

今年年初，AI Agent（智能体）成为整个行业最热的方向。所谓 Agent，简单说就是：不只是回答问题，而是能自主拆解任务、执行步骤、拿到结果的 AI。

这个方向本身没有争议，争议在于：**AI 应该通过什么方式去执行任务？**

OpenAI 的答案是 **Computer Use**：让 AI 像人一样，操控电脑屏幕上的软件——打开浏览器、点击按钮、填写表单。你可以理解为，AI 坐在电脑前，用鼠标和键盘替你干活。

Anthropic（Claude 背后的公司）的答案是 **Coding Agent**：让 AI 通过写代码来解决问题。不是操控界面，而是直接生成一套解决方案——一段程序、一个脚本、一套工具。

两条路，方向完全不同。

---

## Computer Use：一条结构上有缺陷的路

Computer Use 乍看很直觉：人怎么用电脑，AI 就怎么用电脑。

但这条路有几个根本性的问题。

**第一，它在对着一个不稳定的表面操作。** 网页改版了，按钮换位置了，软件更新了，AI 之前学会的操作就失效了。界面是为人类设计的，它随时都在变，AI 要追着变化跑，永远跑不完。

**第二，它的速度是人类速度的上限。** 点击、等待页面加载、再点击——这是人类的操作节奏。对计算机来说，这极度低效。

**第三，它不留下任何东西。** AI 帮你点了一堆按钮，任务做完了，但没有留下任何可以复用、可以检查、可以修改的东西。下一次同样的任务，还得从头来。

本质上，Computer Use 是在让 AI 扮演一个使用鼠标的人——而不是在发挥 AI 真正的计算能力。

---

## Coding Agent：为什么它赢了？

Coding Agent 走的是另一条路：绕过界面，直接作用于数据和系统。

代码有几个特质，使它在这件事上有结构性优势。

**稳定性。** 代码调用的是 API（程序接口），不是 GUI（图形界面）。API 是给机器读的，不会因为界面改版而失效。

**速度。** 代码以机器速度运行，不需要等待页面渲染，也不受人类操作节奏的限制。

**可留存性。** 代码是一个制品——可以被读、被改、被复用。今天写的代码，明天还能用，甚至能在它的基础上继续建更多东西。

**过往积累。** 代码这个领域有一个其他领域很难复制的先天优势：几十年来，人类写下的代码几乎都是公开的。GitHub 上有数亿个代码仓库，Stack Overflow 有数千万条问答，每个开源库都有文档和使用示例。这些都是结构化的、高质量的数据。AI 在这个基础上训练，等于站在了几十年人类积累的肩膀上。
相比之下，人类使用软件界面的过程——点了哪里、为什么点——几乎从来不被记录，也不公开共享。Computer Use 要学的那类知识，根本没有等量级的积累存在。

**最重要的，是乘数效应。**
一个开发者用 Coding Agent 写出来的程序，可能被几万、几百万用户使用。Coding Agent 的产出不是服务一个人的一次操作，而是创造了一个可以服务无数人的工具。这是 Computer Use 做不到的——帮一个人点按钮，就只帮了那一个人。

---

## OpenAI 的并入，是一个信号

Codex 并入 ChatGPT，表面上是产品整合，但背后是一次路线上的调整。

OpenAI 年初押注 Computer Use，但这条路没有跑通。与此同时，Coding Agent 在开发者群体里迅速扩散——开发者们用它写代码、搭工具、自动化工作流，效果显著。

把 Codex 并进 ChatGPT 的核心逻辑，是承认：**Coding Agent 才是 Agent 时代的中心，不能再把它放在边缘。**

但这个并入也暴露了一个还没有解决的问题。

ChatGPT 的主要用户是普通消费者，Codex 的用户是开发者，这两类人的需求和使用方式完全不同。很多 ChatGPT 用户对这次合并感到困惑，不知道新功能和自己有什么关系，还担心共用额度会影响自己的正常使用。

OpenAI 随后在网页端保留了独立的额度，这个补丁本身说明：**两个用户群的融合，在产品层面还没有真正想清楚。**

---

## 真正的问题：谁掌握生产侧？

把视野拉开来看，这场竞争的核心其实是：**谁来创造未来的软件？**

过去，软件是由开发者写出来的，再通过互联网分发给普通用户。ChatGPT 这样的产品，是在分发侧建立了一个极大的流量入口——用户每天来这里问问题、获取信息。

但 Coding Agent 作用的是生产侧——它影响的是软件被创造出来的这个环节。

如果未来的大量软件，是由 AI 辅助甚至主导写出来的，那么谁掌握了 Coding Agent 这个环节，谁就掌握了未来数字世界的生产能力。

Anthropic 的路径是：先拿下开发者，开发者用 Coding Agent 产出软件，软件再影响终端用户。这是一条从生产侧出发的扩散路径。

OpenAI 的路径是：用 ChatGPT 的消费者入口出发，试图让 Coding Agent 的能力直接触达普通用户。这是一条从分发侧出发的扩散路径。

两条路最终走向何处，现在还无法判断。

但 Codex 并入 ChatGPT 这件事，至少说明：OpenAI 已经意识到，不能把 Coding Agent 留在边缘。
